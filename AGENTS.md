# Agent Rules

- Avoid vague production-readiness language. Be specific about actual risks, checks, and work completed.
- Do not frame changes around backwards compatibility, migration, or legacy support. ForgeUI is a from-scratch project.
- Keep only Google's A2UI adapter updated for external compatibility expectations.
- Write concise docs, code comments, PR comments, and push comments. Use only the verbosity needed to be clear.
- Act like a software engineer committing code: read context, make scoped changes, verify when practical, and explain outcomes plainly.
- You are not alone in this codebase. Do not revert or overwrite others' edits unless explicitly asked.
